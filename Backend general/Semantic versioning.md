Семантическое версионирование это набор правил, как используются и поднимаются номера версий программного обеспечения.

Версия состоит из трёх частей, которые обновляются в соответствии с типом изменения:
- MAJOR - несовместимые изменения в API
- MINOR - обратно совместимые изменения в API
- PATCH - обратно совместимые исправления багов

В [[REST API]] мажорную версию можно использовать как первую часть пути URL:
```
/api/v1/some/path
```

Также номер версии можно включать в MIME type:
```
GET /orders/xyz HTTP/1.1
Accept: application/vnd.example.resource+json; version=1
...
```

При использовании очередей сообщений версию API можно включать в само сообщение.
